## 現状の本質的な問題

3rd Placeは今、**2つの別々の問題**を同時に抱えている。

| 問題 | 内容 | 緊急度 |
|------|------|--------|
| **ブランド問題** | 宗教っぽさ・透明性不足・見せ方が間違っている | 最高（今すぐ） |
| **ビジネスモデル問題** | サービスの再定義・DB設計・収益化 | 高（Phase 1中に） |

全員が言った通り「**中身はいい、見せ方が間違っている**」。コアを変える必要はなく、パッケージングを変えれば伸びる。

## 2つのレイヤーを分けて考える

```
コミュニティ（3rd Place）
        ↑
    入り口・顔  ← 今ここが「怖い」「宗教っぽい」
        ↓
サービス（3rd Place Hub）
        ↑
    収益・価値提供
```

入り口が怖いせいで、サービスまで人が辿り着かない。まずここを直す。

## サービスの再定義

### 価値提案の言葉を変える

| 今の言い方（NG） | 新しい言い方 |
|----------------|------------|
| 仕事・住居を紹介する | メルボルンで自分の足で立てるよう準備する |
| トライアル確約 | 先輩の情報と繋がりを武器として渡す |
| マッチングサービス | 到着前に答えを持って渡航できるサービス |

### パッケージの提供価値

```
渡航前：
  - 引き継ぎノート閲覧（先輩の生の声）
  - 渡航前Zoomブリーフィング（希望条件ヒアリング）
  - レジュメ添削
  - 3rd Place公認推薦状

到着後：
  - OB連絡機能（現地メンバーへの相談）
  - 1ヶ月フォロー
  - キャッシュバック設計（到着A$10・近況A$5/回・引き継ぎA$30・後任紹介A$50）
```

**12ヶ月滞在の実質コスト：A$85〜135**

## フェーズ別アクションプラン

### 今すぐ（〜4月末）：信頼の土台を作る

#### ブランド修正（コスト0）

**A1. マサさんのLINE発信トーンを変える**
- 「真心」「思い」の長文 → 具体的な情報発信に切り替え
- 例：「今週のイベント情報」「メルボルン生活Tips」

**A2. 公共の場でのイベントを1つ企画する**
- カフェ・公園・スポーツなど誰でも来やすい場所
- 女性一人でも来られる雰囲気の設計
- 初参加者向けウェルカム担当を置く（運営会議参加者10人を活用）

**A3. 一部イベントに少額参加費を導入**
- 「無料=怖い」という心理への対応
- ワンフード制（A$5〜10）で十分
- カレー会は無料のまま維持（まごころ原則）

#### 運営会議10人を即戦力化する

この10人は今すぐ使える最大の資源。役割分担の例：

| 役割 | 内容 |
|------|------|
| イベント担当 | フットサル・文化体験・パブクローリング等の企画・運営 |
| ウェルカム担当 | 初参加者のフォロー・女性が安心できる仕組み |
| DB担当 | 引き継ぎノートの初期データ収集（帰国前メンバーへのヒアリング） |
| 口コミ担当 | Facebook依存を減らすための口コミ・紹介経由の集客 |

### Phase 1（〜6月）：透明性の確立とDB設計

**B1. ウェブサイト/LPの作成**（ダイチ担当）
- 「3rd Placeとは何か」を外から見てわかるようにする
- 運営メンバーの顔出しインタビューを掲載
- 親御さんが見ても安心できるコンテンツ
- 自前の集客チャネルとして育てる

**B2. SNS発信の強化**（みなと担当）
- Instagram・Facebookへの定期投稿
- 写真・動画コンテンツ
- Facebook依存からの脱却を目指す

**B3. 新規参加者の属性入力を必須化**
- LINEグループ参加時にGoogleフォームで1分アンケート
- データとして蓄積

**B4. 引き継ぎノートの入力フロー設計**
- 帰国前1〜2ヶ月のタイミングでフォームを送る
- 構造化されたフォーム（自由記述にしない）
- 店舗名は伏せ、エリア・業種・雰囲気で表示
- 掲載前に先輩が店に許可取得
- マイナス情報は運営が精査して削除
- インセンティブはキャッシュバックより「次の人への贈り物」という文化設計

**B5. 信頼性の裏付け**
- 資格取得の検討（留学カウンセラー等）
- 法人化の準備（日本に帰ったら）

---

### Phase 2（7〜9月）：マネタイズ開始

**C1. メルボルン着パッケージの有料販売開始**（A$150〜200）

**C2. OB連絡機能のローンチ**
- 現地で働くメンバーへのOB訪問的相談機能
- 動機設計：ペイフォワード構造（パッケージ購入時の誓約）
- チームがZoomをセッティングする形で運用

**C3. 東京での集客強化**
- りょうさん（ウィンウィン）との共同イベント
- 「オーストラリア人が来るイベント」という切り口（日本で日本人を集める突破口）
- りょうさんの実家カフェ+バーを会場として活用
- オンラインではなく対面にこだわる

**C4. BtoB告知枠の営業**（A$50〜200/回）
- SIM・海外保険・語学学校向け
- LINEグループ300人・Facebookチャネルへの告知

---

### Phase 3（10月〜）：スケールとDB活用

- 登録店舗30店・求職者200人でタイミングマッチングが機能し始める
- OB連絡機能がコミュニティの循環として定着
- 住居バトンパス仲介の本格展開
- 東京イベントとメルボルンのZoom接続（エル会との連携）

**KPI目標（3ヶ月/12ヶ月）**
- 求職者：30人 / 200人
- 登録店舗：5店 / 30店
- マッチング：10件 / 100件
- 月収：A$200 / A$2,000〜3,000

## 絶対原則（変えない）

1. エージェントに依存しない（OEM・下請けNG）
2. コミュニティ内にお金を見せない（関係がドライになる防止）
3. 属人性を排除（仕組みとして自走できる設計）
4. カレー会は永続無料（まごころ精神）
5. 教授（マサ氏）の合意なく進めない
6. 情報提供者はメンバー限定（外部開放すると信頼性ゼロ）

## 最優先の3アクション（今週〜来月）

1. **マサさんのLINE発信トーンを変える**（今すぐ・コスト0）
2. **運営会議10人に役割を割り振る**（今週中）
3. **公共の場でのイベントを1つ企画・告知する**（来月中）

この3つだけで「怖い」という印象はかなり変わり、自前の資産がさらに育ち始める。

---

## 結論

> 「中身はめちゃくちゃいい。でも見せ方・伝え方が間違っている」

Facebookという外部チャネルに依存した集客から脱却し、自前のコミュニティ・ウェブサイト・SNSを育てながら、「怖い」という第一印象を払拭することが最優先。その上でサービスとしての3rd Place Hubを段階的に立ち上げていく。
