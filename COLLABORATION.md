# コラボレーションルール

このリポジトリでは、複数人と複数エージェントが並行して作業する。
競合や意図しない変更を避けるため、以下のルールに従うこと。

## Git運用
- `main` で直接作業しない
- 必ず作業ブランチで変更する
- `main` は安定版・統合用ブランチとして扱う
- 作業開始前に `git fetch origin main` を実行し、`origin/main` が進んでいないか確認する
- `main` に統合する前に、最新の `main` を自分のブランチへ取り込む
- PR作成前にも再度 `git fetch origin main` を実行し、必要なら `origin/main` を取り込んでから push する
- 変更はできるだけ小さく分ける

## 担当分担

### UI担当
主に以下を担当する。
- `app/` の画面UI
- `components/`
- `public/`
- 文言、画像、レイアウト、導線

### インフラ担当
主に以下を担当する。
- `lib/` のインフラ系コード
- `app/api/`
- 環境変数
- Vercel設定
- 依存関係
- DB / バックエンド基盤

## 触らないルール
- UI担当は `lib/` のインフラ系、環境変数、Vercel設定、依存関係を勝手に変更しない
- インフラ担当は UI の見た目、文言、画像、レイアウトを勝手に変更しない
- `package.json` や `.env.example` の変更は必要性を明確にして行う
- 担当外の変更を見つけても、勝手に直さない

## 共通ルール
- 競合しそうな共通ファイルを触る前は共有する
- 不明点がある場合は、大きく触らず止める
- 担当外の修正が必要な場合は、先に人間同士で合意してから進める
- エージェントを使う場合も、このファイルのルールに従う

## エージェントへの指示
Claude Code / Codex などのエージェントを使う場合は、作業開始時に以下を伝えること。

- `COLLABORATION.md` に従うこと
- 自分が UI担当かインフラ担当か
- 担当外のファイルは勝手に変更しないこと
- `main` では直接作業しないこと
- 作業開始前とPR前に `git fetch origin main` で最新の `origin/main` を確認すること
