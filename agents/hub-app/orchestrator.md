# Hub App オーケストレーター｜統合レポート生成

## 役割
6体のエージェント（品質レビュー3体＋改善提案3体）の結果を統合し、
**現状の品質スコア**と**次にやるべきアクション**を1つのレポートにまとめる。

## 対象ファイル
`hub-app/3rdplace_app.html`（単体HTMLアプリ）

## 実行手順

### Step 1: エージェントを並列実行
以下の6体を**並列**で実行し、それぞれの出力を収集する：

**品質レビュー（今あるものを見る）**
- Agent 1: UX & デザインレビュアー → `agents/hub-app/agent1-ux-design.md`
- Agent 2: データ＆コンテンツレビュアー → `agents/hub-app/agent2-data-content.md`
- Agent 3: 購入者体験シミュレーター → `agents/hub-app/agent3-buyer-sim.md`

**改善提案（より良くする）**
- Agent 4: デザイン改善エージェント → `agents/hub-app/agent4-design-improve.md`
- Agent 5: コンテンツ強化エージェント → `agents/hub-app/agent5-content-enhance.md`
- Agent 6: 機能アイデアジェネレーター → `agents/hub-app/agent6-ideas.md`

### Step 2: 統合レポートを生成
各エージェントの出力を以下のフォーマットに統合する。

### Step 3: レポートを保存
`agents/hub-app/report.md` に出力する。

## 統合時のルール
1. **複数エージェントが同じ課題を指摘** → 優先度を上げる
2. **エージェント間で意見が矛盾** → 両論併記し、判断をユーザーに委ねる
3. **スコアは各エージェントの出力から転記**（改ざんしない）
4. **🔴（今すぐ）は最大3つ**に絞る — 多すぎると何も進まない
5. **実装コストの定義**に従う:
   - 小: CSS/HTML変更のみ、1時間以内
   - 中: JavaScript追加が必要、半日〜1日
   - 大: 新しいデータ構造・外部API・バックエンドが必要
6. **CLAUDE.mdの制約を遵守**: 求人斡旋NG、カレー会有料化NG、口コミサイト化NG
