# 3rd Place Hub App 改善レポート

**実行日**：2026-04-09
**対象**：hub-app/3rdplace_app.html

---

## 総評

単体HTMLアプリとしては非常に高い完成度を持つプロトタイプ。「知る→探す→引き継ぐ」の3層構造は明確で、業界ガイドと引き継ぎノートのコンテンツ品質は「お金を払う価値がある」レベルに達している。一方で、**フォームバリデーションの欠如**（空データ送信可能）、**ARIA属性ゼロ**（アクセシビリティ未対応）、**hex直書きの散在**（デザインルール違反）という技術的な品質課題がある。また、ユーザー体験の最大のギャップは「情報を見た後の次のアクション導線がない」こと — 「この店で働きたい」「この家に住みたい」と思った後に何をすればいいかがアプリ内に存在しない。

## Part B: 改善提案（次にやること）

### Agent 5: コンテンツ強化案（Top 3）

1. **到着後1週間チェックリスト（日別タスク付き）** — Day 1: SIM・銀行 → Day 2: TFN → Day 3: レジュメ作成 → Day 4-5: レジュメ配り → Day 6-7: 家探し。留学エージェントにはない「生活立ち上げロードマップ」

**新業種追加提案**: ファーム（88日ビザ需要）、ホスピタリティ（バー・パブ、高時給）、フードデリバリー（つなぎ）、リテール（ユニクロ・ダイソー等）

## エージェント別詳細所見

### Agent 1｜UX & デザインレビュアー

**総評**
全体として、単体HTMLファイルとしては非常に高い完成度を持つプロトタイプ。タブ導線・モーダル・フォームの基本的なUXフローは整備されており、デザイントークンも概ね適切に運用されている。しかし、アクセシビリティの欠如（ARIA属性ゼロ）、hex直書きの散在、フォームバリデーション不足、モバイルでのフィルターバーの使い勝手など、プロダクション品質に引き上げるには複数の改善が必要。

**UXフローの評価**

- タブ遷移: 4/5 — 流れは自然。Feature Cardから対応タブへの連携も良い。ただしタブ切り替え時にscrollTo(0,0)がない
- モーダル体験: 4/5 — モバイルボトムシート/デスクトップセンター配置。背景クリック・ESC・Xボタンの3つの閉じ方。フォーカストラップなし
- フォーム体験: 3/5 — 5ステップのプログレス表示あり。必須項目のバリデーションが一切ない。空のまま送信可能
- フィルター: 3.5/5 — 適切なフィルター項目。fMinStayに演算子優先度バグ。リセットボタンなし
- 地図: 3.5/5 — Leaflet + CartoDB Voyager。ステータス別ピン色分け + 凡例。マーカークラスタリングなし
- モバイル: 3.5/5 — 600pxブレークポイント。ナビの横スクロールに視覚的手がかりなし

**デザイン品質の評価**

- トークン一貫性: 3.5/5 — hex直書き多数（`.guide-hero`の#0F1A38、`.warning-block`の#FFF8F5、`.fav-btn`の#E84393、`.handover-hero`の#1a3a2a/#2a5a3a、JS内pinColor()、avatarColors等）
- シャドウ・角丸: 4/5 — 4段階シャドウ適切。`--sh-h`が`--sh-lg`と同値、`--rs:10px`が`--r-sm`と近い重複あり
- タイポグラフィ: 4.5/5 — Noto Serif JP/Sans JPの使い分け正確。line-height:1.7で読みやすい
- コントラスト: 4/5 — `--text-3: #8C8278`が小サイズでWCAG AA未達。`.guide-step-desc`のrgba(255,255,255,.55)も低コントラスト
- トランジション: 4.5/5 — `--duration`と`--ease`でほぼ統一。`.btn-form:hover`のfilter:brightnessだけ異なるアプローチ
- アクセシビリティ: 2/5 — focus-visible実装済み、prefers-reduced-motion対応済み、プリントCSS対応済み。ARIA属性ゼロ、フォーカストラップなし

### Agent 4｜デザイン改善エージェント

**リサーチ結果サマリー**

- コミュニティ系は「Soft UI Evolution」が推奨。Social Proof重視
- 「Nature Distilled」スタイルが最適合。テラコッタ・サンドベージュの温かみ
- 現在のNoto Serif JP + Noto Sans JPペアリングは「Japanese Elegant」として最適解。変更不要
- マイクロインタラクションは50-100ms、UIトランジションは150-300ms
- コントラスト比4.5:1以上、focus-visible必須、色だけに頼らない情報伝達

**改善提案（7件）**

1. ヒーローに疑似要素のグロー装飾 + fadeInUpアニメーション
2. ステータスバッジのglowPulse + カード左ボーダー + お気に入りバウンス
3. ナビタブのアクティブ背景色 + インジケーターのスライドインアニメーション
4. CSS変数の重複解消（--sh-h, --rs削除）+ 統計カードのホバー追加
5. 引き継ぎヒーローのhex直書きをCSS変数化（--sage-dark, --sage-deep追加）
6. フォーカスリング3px化 + 鮮度タグにボーダー追加（色以外の情報伝達）
7. モバイルナビにmask-imageフェードで横スクロール示唆
